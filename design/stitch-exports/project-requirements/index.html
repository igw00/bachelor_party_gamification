# The St. Pete Invitational - Design Requirements ## App Overview A real-time scorekeeping app for a 3-day bachelor party event. - 15 players, 3 teams of 5. - Visual Style: Airbnb-inspired (minimal, premium, #FAFAFA backgrounds, rounded cards, subtle shadows). - Accent Color: Deep Coral or Navy. ## Core Screens 1. **Scoreboard (Home)** - Status bar: "Day 2 of 3", countdown. - Team Cards: Ranked 1-3. Total points, progress bar to card draw, expandable player list. - Player List: Name, points, recent activity (e.g., "cornhole W"). - Groom Icon: Crown/ring icon for the groom. - Card Preview: "2 held · 1 active". 2. **Quick Add Points (Bottom Sheet)** - States: Collapsed (Basic) and Expanded (Advanced). - Basic: Player selection chips, +/- point stepper, description field. - Advanced: Structured inputs for activity results/Captain Bank. 3. **Rules** - Quick-ref cards: Prize Pool, Activity Points, Card Draws. - Collapsible rulebook sections. 4. **Setup Flow** - Player Entry: 15 fields, Groom toggle. - Draw Teams: Shuffle animation, team assignment, renaming. - Build Card Deck: Card list with types (Power Up, Wild Card, etc.). ## Navigation - Fixed bottom tab bar: Scoreboard (left), Oversized + Button (center), Rules (right).
